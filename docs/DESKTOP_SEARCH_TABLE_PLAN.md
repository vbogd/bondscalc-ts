# План: десктопная таблица результатов поиска

## Статус

Документ фиксирует согласованные решения для последующей реализации. Код, зависимости и текущий мобильный интерфейс этим документом не изменяются.

## Цель

Для широких экранов показывать результаты поиска облигаций в виде таблицы, чтобы их было удобно сравнивать и сортировать. На мобильных экранах сохранить существующий карточный список, который является основным сценарием использования калькулятора.

Таблица и карточки используют одни и те же данные поискового запроса MOEX и ведут на ту же страницу калькулятора облигации.

## Представление по ширине экрана

- На мобильных и узких экранах остаются текущие карточки результатов.
- На десктопном брейкпоинте появляется таблица результатов.
- Точное значение брейкпоинта выбрать при вёрстке; ориентир — `lg` / 1024 px, чтобы таблица не появлялась на планшете с недостаточной шириной.
- Оболочка страницы на широких экранах не должна оставаться ограниченной текущими 560 px: ей требуется отдельная десктопная максимальная ширина.
- Таблица помещается в горизонтально прокручиваемый контейнер. Горизонтальная прокрутка допустима и на десктопе: она позволяет не скрывать выбранные пользователем колонки.

## Колонки

### Обязательные

Две колонки всегда видимы, находятся слева и закреплены при горизонтальной прокрутке:

1. **Название** — в этой ячейке также показывается `SECID`.
2. **ISIN**.

Они не отображаются в меню настройки колонок и не могут быть отключены пользователем. Фон закреплённых ячеек должен быть непрозрачным, а слои и тени не должны перекрывать прокручиваемые ячейки.

### Настраиваемые

Пользователь может включать и выключать остальные данные:

- уровень листинга;
- погашение;
- дата купона;
- размер купона;
- ставка купона;
- текущая доходность;
- цена;
- оферта.

Потребуется отдельно определить дефолтный набор включённых колонок. Он должен быть полезным без настройки и не создавать чрезмерно широкую таблицу.

## Меню «Колонки» и сохранение настроек

- Над таблицей размещается кнопка «Колонки».
- Она открывает меню с чекбоксами только для настраиваемых колонок.
- Выбор сохраняется в `localStorage` под отдельным ключом, не связанным с сохранением поискового запроса.
- При повторном запуске приложения сохраняется именно выбор видимости колонок.
- При появлении новой колонки в будущем она получает явно заданное значение по умолчанию, чтобы старые сохранённые настройки не ломали интерфейс.

## Сортировка

Сортировка по колонкам обязательна.

- Заголовки сортируемых колонок интерактивны и показывают направление сортировки.
- Текстовые значения сортируются как текст, даты — хронологически, цена, ставка и доходность — численно.
- Отсутствующие значения всегда располагаются после известных значений.
- Сортировка применяется к уже полученным результатам поиска и не создаёт дополнительный запрос к MOEX.
- Начальное состояние должно сохранять текущий понятный порядок результатов по краткому названию.

Перед реализацией нужно окончательно утвердить два UX-правила: должна ли третье нажатие на заголовок сбрасывать сортировку к исходному порядку и нужно ли сохранять выбранную сортировку между запусками. Рекомендация: сбрасывать третьим нажатием и не сохранять сортировку, в отличие от видимости колонок.

## Компоненты и техническая модель

Использовать:

- `@tanstack/react-table` — headless-модель для сортировки и видимости колонок;
- shadcn/ui `Table` — семантическая разметка таблицы;
- существующие shadcn/ui `Button`, `DropdownMenu` и `DropdownMenuCheckboxItem` — кнопка и меню настройки колонок.

TanStack Table не рисует UI и не добавляет CSS; визуальная часть остаётся в проектных компонентах shadcn/ui. Его стоит применить из-за сочетания сортировки и сохраняемой видимости колонок, а не ради одной разметки таблицы.

Десктопную таблицу следует вынести в отдельный лениво загружаемый модуль. Это сохранит текущий мобильный стартовый бандл: на телефоне загружается только карточное представление, а TanStack Table подгружается для десктопного режима.

## План работ

1. Добавить shadcn/ui `Table` и зависимость `@tanstack/react-table`.
2. Расширить оболочку поиска для широких экранов и определить финальный десктопный брейкпоинт.
3. Вынести данные и определения поисковых колонок в общую модель, чтобы карточки и таблица не расходились в форматировании цены, доходности, дат и отсутствующих значений.
4. Создать лениво загружаемый компонент десктопной таблицы с обязательными закреплёнными колонками «Название / SECID» и ISIN.
5. Реализовать настройки видимости необязательных колонок, загрузку и сохранение настроек в `localStorage`.
6. Реализовать сортировку и правила сортировки для текста, дат, чисел и пустых значений.
7. Сохранить навигацию из результата к `/bond/:secid` и обеспечить доступность управления с клавиатуры.
8. Добавить тесты и проверить в браузере на мобильной, планшетной и десктопной ширине.

## Проверки

- На мобильной ширине показываются только карточки; таблица и меню «Колонки» не видны.
- На десктопной ширине показывается таблица, а карточки не дублируют результаты.
- Название / SECID и ISIN нельзя скрыть и они остаются видимыми при горизонтальной прокрутке.
- Выбор необязательных колонок переживает перезагрузку страницы.
- Каждый сортируемый столбец верно сортирует данные в обоих направлениях; `null` и отсутствующие значения уходят в конец.
- Форматирование цены, доходности, дат, оферты и уровня листинга совпадает с текущими карточками.
- Переход к калькулятору из результата сохраняется.
- Проходят `npm test`, `npm run build` и `git diff --check`.

## Не входит в первую версию

- серверная сортировка и дополнительная загрузка данных MOEX;
- фильтры по значениям колонок;
- изменение порядка колонок перетаскиванием;
- пагинация;
- настройка колонок на мобильной версии.
