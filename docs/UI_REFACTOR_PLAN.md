# План рефакторинга UI на shadcn/ui

## Цель

Привести UI к стандартным компонентам shadcn/ui и убрать текущую кастомную визуальную систему.

Основная идея: сначала получить простой, согласованный, стандартный shadcn-интерфейс без собственной темы, специальных шрифтов и самописных визуальных паттернов. После этого отдельно вернуться к теме, плотности, цветам и финансовым акцентам.

## Что меняем

- Убираем кастомную тему из `src/index.css`.
- Возвращаем стандартные CSS variables shadcn/ui для neutral theme.
- Убираем кастомные цвета вроде `body`, `surface-soft`, `surface-strong`, `semantic-up`, `semantic-down`, `semantic-warning`.
- Убираем явные шрифты `Inter` и `JetBrains Mono`.
- Для чисел используем `tabular-nums`, а не отдельный моноширинный шрифт.
- Сокращаем или удаляем слой `src/shared/ui/FinancialUi.tsx`.
- Переписываем экранные компоненты через стандартные shadcn/ui primitives.

## Что не меняем

- Не меняем доменную логику расчетов.
- Не меняем работу с MOEX API.
- Не меняем маршруты.
- Не меняем формат хранения пользовательских настроек.
- Не добавляем новую дизайн-систему поверх shadcn/ui.
- Не делаем финальную кастомную тему в рамках этого этапа.

## Компоненты shadcn/ui

Уже есть:

- `Button`
- `Input`
- `Badge`
- `ToggleGroup`
- `DropdownMenu`
- `Tooltip`

Нужно добавить или обновить:

- `Field`
- `Card`
- `Alert`
- `Separator`
- `Skeleton`, если понадобится для загрузки

Для полей формы используем паттерн:

```tsx
<Field>
  <FieldLabel htmlFor="buy-price">Цена, %</FieldLabel>
  <Input id="buy-price" className="tabular-nums" />
</Field>
```

React Hook Form для текущего состояния не нужен: форма уже управляется через `useState`, поэтому `Field` можно использовать напрямую как семантическую и визуальную обертку.

## Предлагаемая структура UI после рефакторинга

### `AppShell`

Оставить простой layout:

- `main`
- ограничение ширины
- базовые отступы
- `bg-background text-foreground`

Без кастомных цветов и декоративных поверхностей.

### Поиск облигаций

Переписать на:

- `Input` для строки поиска;
- `Alert` для пустого состояния, загрузки и ошибки;
- `Card` или аккуратный список с shadcn-стилями для результатов;
- `Badge` для уровня листинга и оферты.

Список результатов должен остаться сканируемым на мобильном экране.

### Калькулятор

Переписать на:

- `Button` для навигации назад и внешних ссылок;
- `DropdownMenu` для внешних ссылок;
- `Card` для блоков "Параметры выпуска", "Результаты", "Детализация";
- `Field`, `FieldGroup`, `FieldSet`, `FieldLabel` для формы;
- `Input` для числовых и date-полей;
- `ToggleGroup` для сценария выхода;
- `Alert` для предупреждений и ошибок;
- `Tooltip` для пояснений расчетных показателей.

## Этапы работ

### Этап 1: Подготовить shadcn/ui базу

- Проверить актуальность текущих shadcn-компонентов.
- Добавить недостающие компоненты: `field`, `card`, `alert`, `separator`, при необходимости `skeleton`.
- Убедиться, что импорты и алиасы из `components.json` работают.
- Запустить тесты.

Ожидаемый результат: в проекте есть все стандартные building blocks для переписывания UI.

### Этап 2: Вернуть стандартную тему

- Заменить кастомные CSS variables на стандартные shadcn neutral variables.
- Удалить нестандартные токены цветов.
- Убрать кастомные font-family.
- Оставить базовые правила `body`, `border-border`, `outline-ring/50`.
- Заменить использование класса `.number` на `tabular-nums`.

Ожидаемый результат: приложение использует стандартную тему shadcn/ui.

### Этап 3: Переписать shared UI слой

- Убрать визуально тяжелые обертки из `FinancialUi`.
- Заменить `InputField` на композицию `Field` + `FieldLabel` + `Input`.
- Заменить `ResultPanel` на `Card`.
- Заменить `StateMessage` на `Alert`.
- Решить, нужен ли `BondBadge`; если нужен, оставить только тонкую доменную обертку над `Badge`.
- Сохранить доступность: label/input linkage, aria-label, tooltip descriptions.

Ожидаемый результат: shared UI слой больше не задает отдельную дизайн-систему.

### Этап 4: Переписать `SearchPage`

- Перевести заголовок, поиск, состояния и результаты на shadcn-композицию.
- Убрать локальные кастомные цвета и формы.
- Сохранить текущие пользовательские сценарии:
  - ввод поискового запроса;
  - debounce;
  - состояние до 3 символов;
  - загрузка;
  - ошибка;
  - пустой результат;
  - переход в калькулятор.
- Обновить тесты, если они завязаны на DOM-структуру.

Ожидаемый результат: экран поиска выглядит как стандартный shadcn UI и сохраняет поведение.

### Этап 5: Переписать `CalculatorPage`

- Перевести форму на `Field`/`FieldGroup`/`FieldSet`.
- Перевести блоки результатов на `Card`.
- Перевести предупреждения на `Alert`.
- Оставить `ToggleGroup`, но убрать кастомный pill-style.
- Оставить `DropdownMenu` для внешних ссылок.
- Заменить numeric styling на `tabular-nums`.
- Обновить тесты, если они завязаны на DOM-структуру.

Ожидаемый результат: калькулятор использует стандартные shadcn-компоненты без кастомной темы.

### Этап 6: Проверка и полировка

- Запустить `npm run build`.
- Запустить `npm test`.
- Проверить UI в браузере на мобильной и desktop ширине.
- Проверить, что текст не вылезает из кнопок, карточек и полей.
- Проверить, что длинные названия облигаций и ISIN отображаются корректно.
- Проверить фокусные состояния клавиатурой.

Ожидаемый результат: поведение сохранено, UI стал более согласованным и менее самописным.

## Риски и решения

### Тесты могут сломаться из-за DOM-структуры

Некоторые тесты ищут ближайший `section` или конкретную структуру `dt/dd`. После перехода на `Card` их лучше переписать на пользовательские роли, тексты и label-based queries.

### Стандартная тема может выглядеть слишком нейтрально

Это ожидаемо. Цель этапа - убрать визуальный шум и получить чистую базу. Финансовую тему можно сделать отдельным следующим этапом.

### Цвета доходности и предупреждений

На первом проходе использовать стандартные `destructive`, `muted-foreground`, `foreground`. Отдельные семантические цвета для прибыли, убытка и предупреждений лучше вернуть позже через осознанную тему, если они действительно нужны.

### Числа без моноширинного шрифта

Использовать `tabular-nums`. Это сохраняет выравнивание цифр, но не тащит отдельный визуальный стиль для всех финансовых значений.

## Definition of Done

- В коде экранов нет зависимости от кастомных токенов `body`, `surface-*`, `semantic-*`.
- `src/index.css` содержит стандартную shadcn-тему без кастомных шрифтов.
- Формы используют `Field` и `Input`.
- Основные блоки используют `Card`.
- Состояния используют `Alert`.
- Числовые значения используют `tabular-nums`.
- `npm test` проходит.
- `npm run build` проходит.
- UI проверен вручную на мобильной ширине.
